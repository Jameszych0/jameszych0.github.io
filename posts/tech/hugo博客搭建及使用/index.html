hugo博客搭建及使用

August 8, 2025

#博客

常用 Hugo 命令速查表

hugo new site [项目名] # 创建hugo项目
hugo new posts/xxx.md # 创建文章
hugo -D               # 生成public静态资源
hugo server -D        # 生成public并启动hugo服务

1 为什么选择Hugo #

Hugo由go编写的静态网站生成器，具有以下特点：

  • 加载速度快
  • 部署简单

2 博客配置 #

2.1 创建博客 #

  1. 下载Hugo

    • 访问hugo github releases

    • windows下载版本：hugo_extended_withdeploy_xxxxx_windows-amd64.zip

    • 解压获得 hugo.exe

  2. 创建Hugo项目

    • 将hugo.exe移入想要创建博客的文件夹中，并在该文件夹下打开命令行，输入命令创建hugo文件

    • 命令：hugo new site [项目名]

    • 这里将该命令生成的hugo文件称作主文件夹，以便于下文的描述

    • 将hugo.exe移入主文件夹中

  3. 添加主题

    • 访问Hugo Themes选择自己喜欢的主题，进行下载

    • 这里以typo主题为例，将下载好的主题解压重命名为typo，放到/themes文件夹中

    • 将typo文件夹中的content和hugo.toml复制到主文件夹中

    • 在hugo.toml添加theme = typo

  4. 启动本地预览

    • 在主文件夹中打开命令行启动hugo服务

    • 命令：hugo server -D

到这里，博客成功启动，主题成功加载。

Hugo项目结构示例：

archetypes/      # 文章模板
    default.md 
content/         # 存放文章
public/          # 生成的静态网页
static/          # 静态资源（图片、PDF 等）
themes/          # 主题文件夹
hugo.toml        # 配置文件

2.2 主题配置 #

主题详细配置可参考 Typo Wiki。

以下是一个示例 hugo.toml（可直接套用并根据需求修改）：

baseURL = 'https://你的GitHub用户名.github.io/' # 部署地址
languageCode = 'zh-cn'
title = "我的 Hugo 博客"
theme = 'typo'

# Google Analytics（可选）
googleAnalytics = ""

[taxonomies]
tag = 'tags'
category = "categories"

[params]
description = "个人技术博客"
theme = 'light' # 自动切换暗/亮色
colorPalette = 'catpuccin'
hideHeader = false
showFooter = false
disableImageProcessing = true
faviconPath = ""


homeCollectionTitle = 'Posts'
homeCollection = "posts"


paginationSize = 8 
listSummaries = true
listDateFormat = '2 Jan 2006'

[params.breadcrumbs]
enabled = true
showCurrentPage = true
home = "~"


[[params.menu]]
name = "home"
url = "/"

[[params.menu]]
name = "posts"
url = "/posts"

[[params.menu]]
name = "categories"
url = "/categories"

[[params.menu]]
name = "tags"
url = "/tags"

[[params.menu]]
name = "about"
url = "/resume.pdf"

[markup]
[markup.highlight]
codeFences = true
style = 'github'

[params.giscus]
enable = false

提示：[[params.menu]] 用于设置导航栏，可以配置：

  • categories
  • tags
  • posts中的文件夹
  • static中的文件。

3 博客部署 #

3.1 部署到Github Pages #

将public文件夹提交到github仓库里，通过github page发布。

注：在主文件夹通过hugo -D生成public静态资源文件夹

  1. 创建一个公共github仓库，仓库名{github用户名}.github.io

  2. 前往Setting -> Pages -> Branch选择main分支，然后保存，会自动开启https://{github用户名}.github.io的地址，这地址也是以后访问博客的地址

  3. 将public文件夹提交到刚才创建的仓库中

    git init
git add .
git commit -m "first commit"
git branch -M main
git remote add origin {你的github仓库地址}
git push -u origin main

  4. 上传成功后访问 https://{github用户名}.github.io，成功搭建属于自己的Hugo博客

3.2 Github Action自动化部署 #

博客部署成功后，我们要通过以下步骤来提交博客：

  1. 写完文章后，在主文件夹hugo -D生成public文件夹
  2. 再把public文件夹push到仓库中

这样难免有些繁琐，我们可以通过Github Action实现一步部署。

  1. Github创建一个私有仓库，用于存放Hugo的主文件

  2. 前往Setttings -> Developer Settings -> Personal access tokens，创建一个token(classic)

  3. 为保证安全，将生成的token，保存的仓库的变量中，前往Settings -> Secrets and variables -> Actions中设置

  4. 在hugo主文件创建一个.github/workflows/xxxx.yaml文件

    name: deploy

# 代码提交到main分支时触发github action
on:
  push:
    branches:
      - main

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
        - name: Checkout
          uses: actions/checkout@v4
          with:
              fetch-depth: 0

        - name: Setup Hugo
          uses: peaceiris/actions-hugo@v3
          with:
              hugo-version: "latest"
              extended: true

        - name: Build Web
          run: hugo -D

        - name: Deploy Web
          uses: peaceiris/actions-gh-pages@v4
          with:
              PERSONAL_TOKEN: ${{ secrets.你的token变量名 }}
              EXTERNAL_REPOSITORY: 你的github名/你的仓库名
              PUBLISH_BRANCH: main
              PUBLISH_DIR: ./public
              commit_message: auto deploy

  5. 在hugo主文件创建.gitignore文件，来避免提交不必要的文件

    # 自动生成的文件
public
resources
.hugo_build.lock
# hugo命令
hugo.exe

  6. 将hugo的主文件上传到仓库，上传成功后会触发Github Action，来自动部署你的静态页面

    git init
git add .
git commit -m "first commit"
git branch -M main
git remote add origin {你的github仓库地址}
git push -u origin main

4 博客使用 #

4.1 写文章 #

  1. 分类建议：

    1. 线上通过categories和tags进行分类
      • categories进行文章类型的分类，比如阅读、技术、生活等
      • tags进行文章更细化的分类，比如技术类，可以添加环境部署tag、编程语言tag等
    2. 本地通过文件夹分类
      • content
        • posts
          • 分类1
          • 分类2
        • about.md
        • _index.md

  2. 配置文章模板（archetypes/default.md）：

    +++
title = "{{ replace .File.ContentBaseName `-` ` `  | title }}"
date = {{ .Date }}
draft = true
tags = []
categories = []
toc = false
showTags = true
math = false
hideBackToTop = false
+++

  3. 创建文章

    hugo new posts/分类1/文章名.md # 创建文章

  4. 写完直接提交主文件夹即可（自动部署）：

    git add .
git commit -am "注释"
git push

4.2 多设备写作 #

如果后期需要使用其他电脑进行文章的上传，可以按照本节的描述操作。

  • 在新电脑：

    git clone https://github.com/你的用户名/你的主文件夹仓库.git

  • 写文章前先：

    git pull
