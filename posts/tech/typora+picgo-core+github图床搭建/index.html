typora+picgo-core+github图床搭建

August 10, 2025

#博客

Table of contents

  • 本地存储
  • 图床存储
    • github仓库配置
    • typora配置

​ 我们搭建完博客后，博客图片的存储问题随之而来。两个选择：本地存储、图床存储。

本地存储 #

​ 将markdown文章放在文件目录content/posts下，通过设置typora即可完成。

注：posts文件夹下不能有其他文件夹存放文章，其他文件夹的文章无法显示

图床存储 #

​ 本地存储虽然方便，但是随着图片的增多会导致git仓库变得臃肿，还有一个最重要的原因，本地文章无法进行分类。我的设想是通过文件夹进行分类：

  • posts
    • tech
    • book
    • …

​ 这样的分类方法会导致图片无法正常显示，所以最后我采用了图床存储。

github仓库配置 #

注：图床的选择多样，但不要使用gitee仓库，图片也会无法显示

  1. 创建一个公共项目

  2. 点击此链接申请一个token

    选择No expiration，勾选repo，一定要保存好这个token，后面要用。

typora配置 #

  1. 打开设置，按照下图设置

  2. 点击下载或更新进行picgo-core的安装，找到picgo-core的安装路径，一般为C:\Users\用户名\AppDataRoaming\Typorapicgo\win64注意替换为自己的用户名

  3. 在picgo-core的安装路径下进入cmd操作，安装插件(使图床中的图片按时间分类)

    ./picgo.exe install super-prefix

    若报错NPM is not installed，是没安装node.js环境导致的

  4. 配置github仓库作为图床，通过上图typora界面的打开配置文件开始配置

    {
  "picBed": {
    "current": "github",
    "github": {
      "repo": "userName/repositoryName", // 你的仓库名
      "branch": "main", // 仓库分支，一般为main
      "token": "xxxxxxxxxxxxxx", // 上文申请的token
      "path": "",
      "customUrl": "",
      "origin": "github"
    }
  },
  "picgoPlugins": {
    "picgo-plugin-super-prefix": true
  },
  "picgo-plugin-super-prefix": {
    "prefixFormat": "YYYY/MM/",
    "fileFormat": "YYYYMMDDHHmmss"
  }
}

  5. 配置完成，可以点击typora中的验证图片上传选项以验证是否成功

​ 之后我们在编写文章时，图片就会直接上传到图床里，十分方便。但是由于是使用github作为图床的缘故，typora访问图片可能会无法加载。这个问题我们可以通过设置代理解决。

  1. typora进入偏好设置, 在通用中下拉找到打开高级设置

  2. 在资源管理器中选择conf.user.json，在flag字段添加自己的代理

    /** For advanced users. */
{
  "defaultFontFamily": {
    "standard": null, //String - Defaults to "Times New Roman".
    "serif": null, // String - Defaults to "Times New Roman".
    "sansSerif": null, // String - Defaults to "Arial".
    "monospace": null // String - Defaults to "Courier New".
  },
  "autoHideMenuBar": false, //Boolean - Auto hide the menu bar unless the `Alt` key is pressed. Default is false.
  // Array - Search Service user can access from context menu after a range of text is selected. Each item is formatted as [caption, url]
  "searchService": [
    [
      "Search with Google",
      "https://google.com/search?q=%s"
    ]
  ],
  // Custom key binding, which will override the default ones.
  // see https://support.typora.io/Shortcut-Keys/#windows--linux for detail
  "keyBinding": {
    // for example: 
    // "Always on Top": "Ctrl+Shift+P"
    // All other options are the menu items 'text label' displayed from each typora menu
  },
  "monocolorEmoji": false, //default false. Only work for Windows
  "maxFetchCountOnFileList": 500,
  "flags": [
    [
      "proxy-server",
      "socks5://127.0.0.1:7890"
    ]
  ] // default [], append Chrome launch flags, e.g: [["disable-gpu"], ["host-rules", "MAP * 127.0.0.1"]]
}

​ 这样typora也可以流畅地显示图片了。
